All About Automation

Within the modern developer workflow there are many task which will can take a serious amount of time. Things such a file minification, prefixing your css, and browser compatibity issues are all examples of repetive development tasks. With the use of Gulp, developers can save time by automating the repetive tasks which allows for more productivity.

Gulp is a Javascript task runner that allows developers to automate tasks such as

  • Run a live developement server
  • Compile CSS from Sass or Less
  • Prefix, concatenate, and minify CSS files
  • Fix the structure and layout of code
  • Minify Images
  • And so much more.....

The Flow of Things

In order to impliment Gulp into your next project we must start by defining a specific task we would like to accomplish. Within a task a set of project files can be loaded into the gulp stream to be precessed. Once the files are in the stream we can define automation and build modifications to these files. After all is said and done we can send the new and imporved code to a specified file. For example, if we wanted to use Sass we could define a task, compile the sass into css, minify the css, prefix the css, then send it all to our main css file.

The Gulp API

API Purpose
gulp.task Define a task
gulp.src Reads files in
gulp.dest Writes files out
gulp.watch Watch files for changes
