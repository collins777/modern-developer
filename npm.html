Introduction

Node package manager introduces developers to a whole new world of tools. These tools range from open source packages, libraries, and frameworks. In order to use npm you must first Install Node js. Node js allows developers to write and run JavaScript files on the server. After some type developers began to use Node js to build open sources tools for things such as task automation and compilers to aid with local web developement

In order to access these tools found within Node js, developers must have a tool to manage the open source code. NPM is a simple command line interface which allows developers to intall packages from the Node ecosystem on to their local computers. Currently NPM is the largerest package registry in the world with over 350,000 packages available for use. Now we will discuss how to impliment NPM into your workflow. For the sake of simplicity we will only be covering Macintosh instructions.

Installing Node.js

  1. In order to use NPM in your next project you must first install the latest version of Node.js to your local machine. Here is a link to do so Node.js
  2. Once you have installed Node.js, open your terminal to check that you have properly installed the software. The terminal can be found inside the Applications Folder, then open the Utilites Folder, next you should be able to navigate to the terminal.
  3. In order to test if Node has been properly installed simply insert the command 'node -v' into the terminal. The -v stands for version, after this command is ran the terminal should return the a specified version of Node.js in which you've chosen to install.
